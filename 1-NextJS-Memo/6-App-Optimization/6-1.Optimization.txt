1. Image Optimization
1-1. Local를 통해 사진 객체를 가져오면 해당 사진에 대한 정보가 담겨 있다. => import logo from '../image/logo.png';
1-2. native img 태그는 src={logo.src}로 받아서 경로를 통한 렌더링, Image 컴포넌트는 src={logo} : 사진 객체의 정보를 받아서 렌더링
1-3. 사진 객체 내부에는 src, width, height이 존재한다. => Image 컴포넌트는 이를 받아서 동적으로 각 화면 해상도를 계상해서
이미지 사이즈 조절을 한다.
1-4. 1-3은 srcset이라는 props로 처리가 된다.


2. Image Size
2-1. **Image 컴포넌트는 부여된 높이와 너비를 계상하여 자동으로 리사이징하여 렌더링한다.
2-2. 이 떄 width과 height을 명시적으로 부여해서 파일을 리사이징할 수 있다. (600을 400으로 바꾸면 400을 기준으로 리사이징)
2-3. size props는 여러 가지 값을 주어서 리사이징할 수 있게 해준다. (10vh, 10vw...)


3. priority props
3-1. Image 컴포넌트는 default로 loading = 'lazy'이다. => 사진을 미리 로드하지 않고 페이지에 사진이 필요할 떄 로드한다.
(해당 이미지가 애플리케이션 사용자의 화면에 필요할 떄를 의미한다.)
3-2. priority는 해당 이미지를 우선 로드하고 보여준다. => 필요할 떄가 아닌 언제나 로드된다.
