1. 'use client'가 붙지 않은 컴포넌트는 SSR이다. 클라이언트에게 HTML을 내려준다.

2. 서버 컴포넌트에 대한 요청이 들어오면 서버에서 JS를 실행한다. => DB 접근 Data Fetching, 유효성 검사..
3. 2를 토대로 JSX를 HTML로 변환한다.
4. 클라이언트에게 HTML로 응답한다.

*** 서버 컴포넌트는 Real HTML만 전달하기 떄문에 이후에 JS로직으로 할 수 있는게 없다.
=> 이벤트 핸들러나 hooks 사용이 불가능하다.


5. 'use client'가 붙으면 클라이언트 컴포넌트이다. => CSR이다.

6. 동작 방식
6-1. 클라이언트 컴포넌트 요청이 전달되면 JSX를 HTML로 변환하여 HTML로 생성한다.
6-2. 클라이언트 JS 로직을 JS 번들로 만들어서 HTML과 함꼐 응답한다.
6-3. 브라우저는 전달받은 HTML을 보여주고 JS 번들을 실행한다. 이 작업을 하이드레이션이라고 한다.
(HTML를 일단 보여줌으로써 효율 증대)
6-4. 하이드레이션이 완료되면 이벤트 핸들러나 이후 유효성 검사, 기타 작업이 가능한 클라이언트 컴포넌트가 된다.
