1. getStaicProps, getStaticPaths을 통해서 페이지를 구축한다.
=> 이 떄 getStaticPaths를 통해서 동적 경로를 인식하고 각 경로의 data를 getStaticProps를 통해 생성해낸다.


2. 1의 방식으로 사전 렌더링된 HTML은 클라이언트 요청이 전달이 도달하면 클라이언트에게 건네준다
(서버에서 React 컴포넌트의 JSX를 반환하여 HTML을 생성하고 그 HTML이 전달되는 것이다.)


3. HTML이 전달되면 사용자에게 HTML이 보여지고 하이드레이션이 진행된다.


4. 하이드레이션이란?
4-1. 정적 HTML 페이지에 React 동적 기능을 추가하는 과정을 의미한다.
4-2. 이벤트 바인딩, 상태 관리가 가능해진다.


5. 과정
5-1. 사전 HTML을 전달하여 사용자의 인터페이스에 보여진다.
5-2. HTML의 script 태그에는 Next.js의 js 번들을 요청하는 src 경로가 존재한다. => 이를 요청한다.
5-3. JS 번들이 도착하면 하이드레이션이 진행된다.
(기존의 HTML을 비교하여 가상 DOM으로 구조를 맞추고 이벤트 바인딩을 실행한다.)
5-4. 이 작업이 완료되면 하이드레이션된 가상 DOM이 페이지로써 역할한다.


*** COMEBACK
=> 요청이 다시 전달되면 브라우저에는 HTML이 존재하기 때문에 하이드레이션만 진행하면 된다.
