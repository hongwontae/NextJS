1. SSR
1-1. Next 애플리케이션 서버에서 들어오는 요청을 처리해야 할 필요가 존재 => 서버 사이드 렌더링이 이유
1-2. 이 떄 getServerSideProps를 통해 요청을 처리할 수 있다.
1-3. exprot async function getServerSideProps(){}
1-4. 1-3 함수는 getStaticProps와 함께 사용하면 에러가 발생한다. => 둘 다 props에 영향을 주는 함수이기 떄문이다.


2. getServerSideProps의 인자
2-1. 해당 인자는 params, req, res를 받을 수 있다.
=> NodeJS의 req, res에 동일하다.
2-2. const {params, req, res} = context;


3. getServerSideProps의 특징
3-1. 요청 떄만 실행되어서 getServerSideProps를 실행하여 props를 해당 컴포넌트에 전달
3-2. props를 통해 JSX를 반환하여 HTML 구축하고 클라이언트에게 전달
3-3. 클라이언트는 HTML을 보여주고 하이드레이션 작업 시작
3-4. React 애플리케이션처럼 동작


4. getServerSideProps는 요청 떄마다 새로운 HTML을 전달한다.
