1. layout
1-1. layout.js 이름을 갖춰야 한다.
1-2. export defalut로 내보내야 한다.
1-3. layout 내의 children은 page.js를 의미한다.
1-4. 부모 layout은 자동으로 자식 page.js를 덮는다.
1-5. 기본적으로 서버 컴포넌트이다.
1-6. 레이아웃에서 데이터를 페칭할 수 있다.
1-7. 동일한 인스턴스를 사용하기 떄문에 동일한 layout를 사용하면 state가 공유된다. (남아 있는다.)
1-8. layout.js는 캐시되지 않는다. => 초기 요청시에 해당 컴포넌트 jsx와 결합되어 나가지만 JS 번들에 존재하지 않는다.
=> 다만 partial redering으로 효율적으로 처리된다.


2. root layout
2-1. root layout은 최상단 레벨에 존재해야 하고 <html>과 <body>가 존재해야 한다.


3. Template
3-1. template.js는 레이아웃과 비슷하지만 항상 새로운 인스턴스를 생성하기 때문에 state가 남아 있거나 공유되지 않는다.


4. Tip
=> MainNavigation.js에서 usePathName을 사용해서 링크가 활성화되어 있는지 아닌지를 판단하여 동적 내비게이션을 만들 수 있다.
