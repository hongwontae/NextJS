1. App router/ page router conflict
1-1. App router가 Page Router보다 우선순위에 놓여있지만 하나만 사용해 애플리케이션을 구축하는게 효율적이다.

2. 폴더는 파일 경로를 의미하고 폴더 내의 파일은 UI를 구축하는데 사용된다.

3. 각각의 폴더 Name은 URL Segment이다.

4. 폴더 내부의 파일로 UI, error, 404 처리를 할 수 있고 이를 구분하기 위해 특별한 FileName을 사용한다.
4-1. layout.js : 폴더 내 page.js, 자식의 layout 처리한다.
4-2. page.js : 폴더 내 UI
4-3. loading.js : 폴더 내 page.js, 자식의 page.js의 loading 처리 page
4-4. not-found.js : 올바르지 않은 경로의 page.js, 자식의 page.js 처리
4-5. error.js : 에러가 발생했을 떄 폴더 내 page.js, 자식의 page.js 대신 처리
4-6. global-error.js : 전체 에러 page.js
4-7. route.js : API
4-8. defalut.js : 병렬 라우트에 관한 js


5. app 폴더 내부에 page.js가 아닌 파일이 존재할 수 있다. => 일반 컴포넌트 파일이 존재할 수 있다.
=> 다만 이 떄는 route가 아니기 떄문에 해당 컴포넌트에 url 접근은 불가능하다.
