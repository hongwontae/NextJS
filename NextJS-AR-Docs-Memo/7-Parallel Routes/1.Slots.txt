1. 병렬 라우트
=> 동시에 두 개의 page를 렌더링하는 기능이다.


2. slots
=> 병렬 라우트를 생성하려면 @folderName
=> 병렬 라우트의 page.js는 부모의 layout.js의 props로 표현할 수 있다.
=> 병렬 라우트 이름은 url segment가 아니다.


3. 병렬 라우트 개념
3-1. 병렬 라우트 내부에 page.js만 존재할 떄 layout에 따라 배치된다.
=> 이 떄 url은 부모 라우트를 가르킨다.

3-2. 병렬 라우트 내부 하나는 또 다른 폴더와 page.js가 존재할 떄 나머지 병렬 라우트와 부모 라우트는
 defalut.js가 존재해야 렌더링된다.
=> 이 떄 병렬 라우트 내부의 폴더 segment를 가리키기 떄문에 논리상 나머지 병렬 라우트와 부모 라우트의 page.js가 무시된다.
=> defalut.js로 해결할 수 있다. => 이 떄는 layout이 defalut.js를 가리킨다.


4. 조건부 병렬 라우팅
4-1. 삼항 연산자나 일반 연산자로 라우팅을 선택할 수 있다.


5. 병렬 라우트 안에서 각 에러.js, loading.js를 처리할 수 있다.


6. 병렬 라우터 중 하나의 라우터가 리렌더링되거나 업데이트되어도 다른 하나의 라우터는 원래 상태를 그대로 유지한다.
=> 각 병렬 라우터는 독립적으로 동작하기 때문이다.
