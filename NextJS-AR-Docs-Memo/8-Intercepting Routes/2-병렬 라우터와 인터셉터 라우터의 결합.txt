1. 병렬 라우터는 두 개의 라우터를 동시에 렌더링할 수 있는 기능이다.
=> @ folderName, 해당 병렬 라우터 폴더는 url segment가 아니다.
=> 또한 인터셉트 라우터가 가로챌 라우터를 가리키기 위해 고려해야 할 dot에 해당하지 않는다.
=> 병렬 라우터를 표현하고 싶다면 부모 레벨 layout props에 folderName으로 가져와서 기술해야 한다.
=> 만약 두 개의 병렬 라우터가 존재할 떄 하나의 병렬 라우터 segment로 향하면 다른 하나는 default.js가 렌더링된다.
=> 부모 레벨에서 layout에서 props children이 구현되면 항상 렌더링되어 있다.


2. 인터셉터 라우터는 Clinet-Side-Navigation으로 일반 라우터 페이지에 접근했을 떄 인터셉트해서 대신 렌더링한다.
=> 다만 (dot..)folderName으로 규정하고 folderName을 일치시켜야 한다.
=> 새로고침이나 특정 url segment로 페이지에 접근하면 일반 라우터가 렌더링한다.


3. 병렬 라우트 + 인터셉터 라우터
=> 병렬 라우터와 인터셉터 라우터를 같이 사용하면 인터셉터 라우트가 레이아웃만 가져가서 표현하는게 아닌 다른 하나의 라우트가
남기 떄문에 동적으로 표현할 수 있다.
