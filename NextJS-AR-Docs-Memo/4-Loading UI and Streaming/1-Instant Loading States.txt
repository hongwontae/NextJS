1. Instant Loading States
1. 페이지 전환 시 loading.js를 렌더링한다.
2. loading.js 파일은 사용할 라우트 폴더 내에 위치하고 하위 라우트에서 사용 가능하다.
3. loading.js는 page.js에 http 요청을 보내면 loading.js가 먼저 로드된다.

3. page.js 요청이 오면 loading.js가 suspense 경계를 이룬다.
4. <Suspense fallback={loading.js}>
<PageComponent>
</Suspense> => 이런 형식처럼 되는 것이다.
5. 비동기 페이지가 로드되면 fallback를 치우고 렌더링된다.

6. loading.js가 존재하면 Streming 기능이 활성화된다.
6-1. SSR은 HTML을 클라이언트에게 전달하고 브라우제에서 css, js를 처리한다.
6-2. 스트리밍은 전체 HTML이 생성될 떄까지 기다리지 않고 부분적으로 HTML을 청크 형식으로 받아 부분 렌더링 + 부분 하이드레이션을
진행한다. => 더 빠르고 인터랙티브한 앱이 된다.


7. 스트링밍 METADATA
7-1. MataData가 페칭이 되면(generateMetadata) 스트리밍이 시작된다.
7-2. 요청의 첫 시작인 청크는 메타데이터를 포함한다.


8. loading.js가 끝나고 페이지가 보여지는 시점
=> 데이터가 준비되고 HTML이 전달되고 스트리밍되고 하이드레이션이 끝나는 시점이 페이지가 보여지는 시점이다.
